  • iframe 透传全局对象：
    爬取目标的数据（如 帆软报表）被封装在复杂的 Frame 闭包中，且接口参数涉及复杂的动态加密（Token/Ticket），直接模拟请求难度大且易失效

    然后利用 page.evaluate 穿透 Frame 边界，直接访问子 Frame 的 window 全局对象（如 window.FR.SessionMgr）提取关键 SessionID 或数据对象，从而进行模拟数据请求，获取真实的电力现货出清，负荷预测等数据

    多个定时任务可能同时触发（同一时刻被 node-schedule 监听），并发执行会导致浏览器 Session 冲突、Cookie 覆盖
    为每个任务定义 6 种互斥状态（scheduled 已计划 → queued 排队中 → running 执行中 → completed/failed/cancelled 终态） 通过 循环检测锁状态，确保同一时刻仅有一个任务处于 running 状态

    请求级重试：
    触发条件：HTTP 500/502 等服务器错误，使用退避策略
    第一次失败后等待较短时间（如 1 秒） 第二次失败后等待更长时间（如 2 秒） 依次递增，避免瞬时大量请求压垮服务器
    Axios 拦截器（Interceptor）自动捕获错误并重试
    任务级重试：

    页面上下文重载：恢复 DOM 状态和业务执行环境
